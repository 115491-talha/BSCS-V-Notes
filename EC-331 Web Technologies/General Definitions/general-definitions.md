## **EC-331 — Web Technologies (General Definitions with Examples)**

### **1. Web Basics**

* **Web Application:** A program that runs on a web server and is accessed through a browser.
  *Example:* Gmail or Facebook.

* **TCP/IP (Transmission Control Protocol / Internet Protocol):** The main protocol suite that enables communication over the internet.
  *Example:* When you open a website, TCP/IP transfers data between your computer and the server.

* **Web Server:** Software or hardware that serves web pages to clients.
  *Example:* Apache, Nginx, or Microsoft IIS.*

* **Virtual Hosting:** Hosting multiple websites on one physical server.
  *Example:* One server hosting both `abc.com` and `xyz.com`.*

* **Chunked Transfer:** Method of sending data in pieces without knowing the full size beforehand.
  *Example:* Streaming a large video file online.*

* **Caching:** Storing copies of web resources to speed up loading.
  *Example:* Browser cache storing images for faster page reload.*

* **Extensibility:** Ability to add new features or modules to a system without changing its core.
  *Example:* Adding plugins to a web server.*

---

### **2. Markup and Styling Languages**

* **SGML (Standard Generalized Markup Language):** A meta-language used to define other markup languages.
  *Example:* HTML and XML are based on SGML.*

* **HTML5 (HyperText Markup Language):** The latest version of HTML for structuring and presenting web content.
  *Example:* Used to create web pages with videos and forms.*

* **CSS3 (Cascading Style Sheets):** A styling language that controls layout, color, and design of web pages.
  *Example:* Changing font color and background of a webpage.*

---

### **3. XML and Related Technologies**

* **XML (Extensible Markup Language):** A markup language for storing and transporting data in a structured format.
  *Example:* Config files or RSS feeds.*

* **XHTML (Extensible HTML):** A stricter version of HTML that follows XML rules.

* **XHTML-MP (XHTML Mobile Profile):** A lightweight version of XHTML used in mobile devices.

* **WML (Wireless Markup Language):** Used for creating pages on old mobile browsers (WAP).

* **XSL (Extensible Stylesheet Language):** Used to transform and style XML documents.

* **XML Query Language (XQuery):** A language for querying XML data.
  *Example:* Extracting specific data from an XML file.*

---

### **4. Web Services and Communication**

* **Web Service:** A software system that allows communication between applications over the web.
  *Example:* A payment API like PayPal.*

* **SOAP (Simple Object Access Protocol):** XML-based protocol for exchanging structured information.

* **REST (Representational State Transfer):** An architectural style using standard HTTP methods (GET, POST, PUT, DELETE).
  *Example:* REST API used to fetch user data from a web app.*

* **Processing HTTP Requests:** The server receives data sent by the client.
  *Example:* Login form submission.*

* **Processing HTTP Responses:** The server sends data back to the client.
  *Example:* “Login successful” message.*

* **Cookies:** Small data files stored in the browser for session tracking.
  *Example:* Remembering a user’s login.*

* **P3P (Platform for Privacy Preferences):** Protocol that communicates a website’s privacy practices to users.

* **Dynamic Content Delivery:** Generating web content on the fly based on user input or data.
  *Example:* Personalized news feed.*

---

### **5. Server Configuration and Security**

* **Server Configuration:** Setting up a web server’s parameters such as port, root directory, and access control.
  *Example:* Configuring Apache’s `httpd.conf` file.*

* **Server Security:** Measures to protect data and prevent unauthorized access.
  *Example:* Using HTTPS certificates and firewalls.*

---

### **6. Web Browsers and Architecture**

* **Browser:** A client application used to view web pages.
  *Example:* Google Chrome or Firefox.*

* **Browser Architecture:** Internal components that process and render web content — networking, rendering engine, and JavaScript engine.

* **Rendering Engine:** Converts HTML/CSS into a visible web page.
  *Example:* Blink engine in Chrome.*

---

### **7. Active Browser Pages (Client-Side Technologies)**

* **JavaScript:** A scripting language that enables interactivity in web pages.
  *Example:* Showing a pop-up message when a button is clicked.*

* **DHTML (Dynamic HTML):** Combination of HTML, CSS, and JavaScript to create dynamic pages.

* **AJAX (Asynchronous JavaScript and XML):** Technique for loading data without reloading the page.
  *Example:* Live search suggestions on Google.*

* **JSON (JavaScript Object Notation):** Lightweight data format for exchanging information between client and server.
  *Example:* Sending user data in APIs.*

---

### **8. Web Application Development Approaches**

* **Client-Side Development:** Processing and rendering done in the user’s browser.
  *Example:* JavaScript frameworks like React.*

* **Server-Side Development:** Processing done on the server before sending the response.
  *Example:* ASP.NET, PHP, or Node.js.*

* **MVC (Model-View-Controller):** Design pattern that separates application logic, UI, and data.

* **Scripting Language:** Interpreted programming language for automating web tasks.
  *Example:* JavaScript, Python, or PHP.*

---

### **9. Search and Optimization**

* **Search Technologies:** Methods for retrieving information from the web.
  *Example:* Google’s web crawlers and indexing.*

* **Search Engine Optimization (SEO):** Techniques to improve website ranking on search engines.
  *Example:* Using keywords in page titles and meta tags.*

---

### **10. Advanced and Emerging Web Topics**

* **Semantic Web:** Web of data that allows machines to understand information contextually.
  *Example:* Google Knowledge Graph.*

* **Future Web Frameworks:** Next-generation tools for efficient web development.
  *Example:* Angular, React, or Vue frameworks.*
